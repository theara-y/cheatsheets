Object Oriented Programming

A methodology or paradigm to design a program using classes and objects. It simplifies software development and maintenance by providing some concepts.

Procedure Oriented Programming

Object

An object can be anything, tangible or intangible. An object can contain state (attributes) and behavior (methods). An object can also be defined as an instance of a class, where the object has been initialized and is currently being tracked in computer memory.

Classes

A class is a computer document that defines the object; including attribute, methods, initialization logic, and more. It is used as a blueprint to then create instances when the program is run.

Constructors

A constructor is a special method within a class which is run when an instance is being created. It is used to perform initialization tasks for the object.

Inheritance

Inheritance is a concept in OOP where an object acquires traits(state and behavior) from another a parent object through a parent-child relationship. Inheritance provides benefits such as code reuse, specialized objects and method overriding. Inheritance is also described as a (child) IS-A (parent) relationship.

Types of Inheritance

  • Single
  • Multi-level
  • Hierarchical
  • Multiple
  • Hybrid

Polymorphism

Polymorphism is a concept in OOP that focuses on the ability of changing and being adaptive. Examples of polymorphism include using a parent class reference to refer to a child class, method overriding and overloading.
